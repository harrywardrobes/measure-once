# Action Handler Config Blocks

Handler config blocks are the inline form panels that appear inside the **Add action** /
**Change action** editor modal on the [Action Handlers admin tab](/admin#tab-action-handlers).
Each block is rendered when an admin selects a particular handler type from the type
dropdown, and it lets them configure the behaviour that fires when an operator clicks the
corresponding card-action strip on the customer pipeline board.

There are three handler types with config blocks. Each maps to its own config block component in
`src/react/pages/admin/HandlerConfigBlocks.tsx`.

---

## Handler types at a glance

| Handler type | Component | Config keys | Side effects when clicked |
|---|---|---|---|
| **Schedule visit** | `ScheduleVisitConfig` | `visitType`†, `defaultDurationMin`\* | Opens the scheduling modal pre-set to the chosen visit type and duration |
| **Show message** | `ShowMessageConfig` | `title`\*, `message`† | Displays a read-only popup — no API call, no status change |
| **Start design visit wizard** | `StartDesignVisitConfig` | `defaultDurationMin`\*, `intermediateLeadStatus`\*, `submittedLeadStatus`\*, `termsAndConditions`\* | Opens the multi-step design visit wizard; sets lead status on open and again on submit (see two-phase flow below) |

\* Optional field — the block still saves and works if this value is left blank.  
† Required field — `visitType` must be one of `design`, `survey`, or `other`; `message` cannot be left blank.

---

## Two-phase status flow (StartDesignVisit only)

`StartDesignVisitConfig` is the only handler type with two lead-status selectors. This
reflects the two distinct moments in the design-visit lifecycle where the contact's
pipeline status should change:

1. **In-progress status** (`intermediateLeadStatus`) — applied the moment the operator
   opens the design visit wizard. Use this to mark the contact as "design visit in
   progress" so other team members can see work is underway.

2. **Submitted status** (`submittedLeadStatus`) — applied when the operator submits the
   completed design visit form. This can be a top-level lead status **or** a sub-status;
   the dropdown groups both options under separate headings.

Both fields are optional. You can configure either, both, or neither depending on your
pipeline conventions.

## ShowMessage

The simplest handler type — no scheduling, no status change. The admin writes a plain-text
message (required, ≤2000 chars) and an optional popup title (≤120 chars). When an operator
clicks the card-action strip, the message is shown verbatim in a popup dialog and nothing
else happens.

## StartDesignVisit

The most configurable handler type. In addition to the two-phase status fields described
above, the admin can set a default duration (minutes) for the visit and paste in optional
Terms & Conditions text (≤4000 chars, shown to the customer on the sign-off page).
