# FileUploadField

A MUI-styled file input that shows upload progress, success, and error states inline.
Wraps a hidden `<input type="file">` with a Browse button, a filename chip, a progress
bar, and status adornments.

## When to use

- Wherever a user needs to attach a file to a form — profile photos, document uploads,
  design drawings.

## When not to use

- For dragging multiple large files — the component supports `multiple` but the UX is
  designed for 1–3 files; a dedicated dropzone is better for bulk uploads.

## Do / Don't

**Do:** set `accept` to restrict the file types to exactly what the server expects.
Show the expected formats in `helperText` (e.g. *"PDF, DWG — max 10 MB"*).

**Don't:** leave `helperText` empty — users need to know size and format constraints
before choosing a file.

Use `onStatusReset` + `resetDelay` to automatically clear the success/error state
after a short delay rather than leaving it frozen.
