Add an on-page search bar to the Customers page so that a search started from the
top navbar can be seen and cleared without leaving the page.

## Context
- File: `src/react/pages/CustomersPage.tsx`
- The page already holds search state seeded from the `?q=` URL param:
  `searchInput` / `search` (around line 942), with a debounce effect (~line 1225)
  that pushes `searchInput` into `search` + resets to page 1, and a URL-sync
  effect (~line 1238) that writes `q` back to the URL.
- Today nothing on the page renders a search input — search only arrives via the
  navbar (CommandPalette builds `/customers?q=...` and navigates). So after a
  navbar search there is no way to view/edit/clear the query from the page.
- The "Sort by" row is the `<Stack direction={{ xs: 'column', sm: 'row' }}>` at
  ~line 1633, containing `<SortSelect>` (left) and the "Show all" toggle group
  (right).

## What to build
Add a search `TextField` into that same Sort-by row.

Requirements:
1. **Placement:** make it the far-right item on the row — to the right of the
   "Show all" toggle group.
2. **Width:** compact, fixed width (~220px, `size="small"`). Do NOT flex-grow.
3. **Pre-filled:** bind its value to the existing `searchInput` state so it shows
   the current query (e.g. when arriving from a navbar search via `?q=`).
4. **Live filter as you type:** typing calls `setSearchInput(...)`, which already
   feeds the existing debounce → `setSearch` + `setPage(1)`. Do not add a second
   debounce; reuse what's there.
5. **URL sync:** rely on the existing effect that writes `q` to the URL so the
   search survives refresh and is shareable — no extra wiring needed beyond
   driving `searchInput`/`search`.
6. **Clear (X):** show a trailing clear button (MUI `IconButton` with the
   already-imported `ClearIcon`, inside an `InputAdornment`) only when there is
   text in the box. Clicking it clears the search ONLY: set `searchInput` to ''
   (and `search` to '' / `setPage(1)` so results reset immediately); `q` drops
   from the URL via the existing sync. Leave sort, stage, and lead-status filters
   untouched.
7. **Placeholder:** `"Search customers…"` when empty. Add a leading `SearchIcon`
   start adornment (both `SearchIcon` and `ClearIcon` are already imported).
8. **Mobile:** the row already switches to `direction="column"` on `xs`. Keep the
   search bar in source order (it stacks after Sort by / Show all). No special
   mobile reordering or full-width treatment required.

Use `size="small"` to match the existing controls and keep styling consistent
with the surrounding MUI components on the page.

## Verify
- Search from the navbar → land on `/customers?q=...` → the new bar is pre-filled
  with that query and the X is visible.
- Type in the bar → list filters live (debounced) and the URL `q` updates.
- Click the X → query clears, full list returns, `q` drops from the URL, X hides;
  sort/stage/lead-status filters are unchanged.
