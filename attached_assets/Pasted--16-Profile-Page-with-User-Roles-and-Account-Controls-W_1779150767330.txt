#16 - Profile Page with User Roles and Account Controls
What & Why
Create a dedicated profile page accessible via the header profile icon. It consolidates account controls (sign out, admin link, connection statuses) that currently clutter the header, and introduces a per-user profile with personal info (job role, privilege level, etc.) that only admins can edit.

Done looks like
Tapping/clicking the profile icon in the top header navigates to a Profile page
The Profile page shows the user's avatar, name, and email (from Replit Auth)
The Profile page shows the user's personal info: job role and privilege level (stored in the database)
Admins see an "Edit" option on any user's profile info; non-admins see it as read-only
Google connection status (connect/disconnect) is on the Profile page
HubSpot connection status dot is on the Profile page
Sign Out button is on the Profile page
Admin link is on the Profile page (visible to admins only)
The top header no longer shows any of the above items — just the profile avatar icon and the refresh button
The Profile page is not in the bottom navigation bar (accessed via profile icon only)
Privilege levels are defined (e.g. Viewer, Member, Manager, Admin) and stored per user
Admins can update any user's job role and privilege level from the admin panel or the user's profile page
Out of scope
Users editing their own name, avatar, or email (these come from Replit Auth and are not editable)
A full user-management list page (just editing via the existing admin panel or an individual profile)
Enforcing privilege levels as access gates on existing routes in this task (stored and displayed, enforcement is future work)
Steps
Extend the users table — Add job_role (text) and privilege_level (text, e.g. 'viewer' | 'member' | 'manager' | 'admin') columns to the existing users PostgreSQL table. Default existing users to 'member'.

Add profile API endpoints — Add GET /api/users/:id/profile to return profile info for a given user, and PATCH /api/users/:id/profile (admin only, using requireAdmin) to update job_role and privilege_level.

Add Profile tab panel to HTML — Add a #tab-profile section to index.html with DOM placeholders for avatar, name, email, job role, privilege level, connection statuses, admin link, and sign-out button.

Wire profile icon as nav link — Make the profile avatar in the header call setTab('profile') on click. Ensure setTab handles 'profile' and toggles the correct panel.

Populate profile tab dynamically — On tab open, fetch the current user's profile from the new endpoint and render all fields. Show an inline "Edit" form for job role and privilege level if the current user is an admin, saving via PATCH /api/users/:id/profile.

Strip header of moved items — Remove Google connect/disconnect, HubSpot status dot, admin link, and sign-out from the #auth-status header render, leaving only the avatar icon.

Style the profile page — Use the app's existing Tailwind utility classes to give the profile page a clean card layout with clear sections for identity, personal info, integrations, and account actions.
