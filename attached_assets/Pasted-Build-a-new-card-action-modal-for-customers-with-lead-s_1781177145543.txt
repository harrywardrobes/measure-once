Build a new card action modal for customers with lead_status = OPEN_DEAL.

## Context (existing code to reuse — do not rebuild these)
- Modals live in `src/react/components/modals/` and are dispatched via `openCardActionModal()` in `src/react/utils/cardActionModalRegistry.ts`, hosted by `src/react/components/CardActionModalsHost.tsx`. Register the new modal there and wire it as the card action for lead_status = OPEN_DEAL (via the card action handlers system / admin ActionHandlersPage).
- Lead statuses are rows in the `lead_status_config` table, seeded from LEAD_STATUS_STAGE_SEEDS in `server.js`. `DEPOSIT_INVOICE` already exists. Status changes must go through the existing lead-status update path (which syncs hs_lead_status to HubSpot).
- QuickBooks integration is in `quickbooks.js` (OAuth tokens in `qb_tokens`, send rate-limiting in `qb_send_log`, existing endpoints for invoice get/update/pdf/send). Estimates are created from design visits in `design-visits.js` and linked via `design_visits.qb_estimate_id`, `qb_estimate_doc_num`, `qb_estimate_history`.
- Email templates live in `email-templates.js` (TEMPLATE_DEFS) backed by the `email_templates` DB table and editable in the admin email templates UI. Emails send via the existing transport (`createMailTransport()` in `auth.js`) using `getEmailTemplate()` + `renderEmail()`.
- Photo uploads already exist (`UploadPhotosModal`, `design-visit-uploads.js`).

## 1. Reusable contact details header
Extract a reusable `ModalContactHeader` component (new file in `src/react/components/modals/`) showing customer name, phone, email, and address — matching the header styling our existing action modals already use. Use it at the top of this new modal, and refactor at least `ContactCustomerModal` and `ArrangeVisitModal` to use it too, so it is genuinely shared rather than duplicated.

## 2. The modal: OpenDealActionModal
Below the contact header, present three options the user chooses between:

### Option 1 — Make amendments (lead status stays OPEN_DEAL)
A hub screen with buttons that open the EXISTING flows (do not rebuild them):
- Upload new photos → opens the existing UploadPhotosModal flow.
- Amend the design → opens the existing design visit edit flow.
- Amend the estimate/invoice in QuickBooks → opens the existing QuickBooks estimate/invoice edit flow.
Important: when an estimate is amended and the existing logic replaces it with a new estimate (the non-editable fallback in `design-visits.js` that logs the old ID to `qb_estimate_history`), the superseded estimate must now ALSO be marked declined in QuickBooks (sparse update TxnStatus = "Rejected"). Add that to the existing replacement logic.

### Option 2 — Accept deal (lead status → DEPOSIT_INVOICE)
Step A — pick the estimate: fetch ALL of this customer's QuickBooks estimates (query by CustomerRef, plus any IDs stored on their design visits) and show them in a list (doc number, date, total, TxnStatus). The user explicitly picks which estimate is being accepted. Offer checkboxes to mark the other open estimates as declined (TxnStatus = "Rejected") at the same time — note in the UI that this should be confirmed with the customer.

Step B — confirmation safety screen (this guards against sending the wrong invoice):
- Show customer name, email, estimate doc number, estimate total (incl. VAT), and the computed 10% deposit amount in £.
- Cross-check the modal contact's email against the QuickBooks customer's email and show a prominent warning if they don't match.
- Require an explicit confirmation checkbox ("I've checked this is the correct customer and estimate") before the send button enables.

Step C — on confirm:
1. Mark the chosen estimate Accepted in QuickBooks (sparse update TxnStatus = "Accepted").
2. Create a deposit invoice exactly like QuickBooks' manual "Convert to invoice → How much do you want to invoice? → Percentage → 10%" flow: a progress invoice linked to the estimate (invoice lines with LinkedTxn to the estimate lines at 10%). Progress invoicing must be enabled in the QBO company settings — detect and surface a clear error if it isn't. If line-level percentage linking fails, fall back to a single-line invoice "Deposit — 10% of Estimate #<docNum>" for round(TotalAmt × 0.10, 2) with a transaction-level LinkedTxn to the estimate, so the total is always 10% of the VAT-inclusive estimate total.
3. Send that invoice through QuickBooks' own templated email using the existing invoice send endpoint (respect the existing qb_send_log rate limit and per-user advisory lock).
4. Send a follow-up email from the app to the customer using a NEW email template (see §4): "deposit invoice sent" message.
5. Move lead status to DEPOSIT_INVOICE via the existing status update path.
6. Store the created deposit invoice ID/doc number against the customer/design visit so it's traceable.
If any step fails, stop, show exactly what succeeded and what didn't, and do not advance the lead status unless the invoice was actually created and sent.

### Option 3 — Not interested (lead status → DECLINED_DEAL)
- Show the customer's open estimates and mark them declined in QuickBooks (TxnStatus = "Rejected"), with a confirmation step first.
- Then OFFER (optional, skippable) sending a thank-you email to the customer using a new template (see §4).
- Move lead status to the NEW status DECLINED_DEAL.

## 3. New lead status
Add DECLINED_DEAL to `lead_status_config` (migration + seed alongside LEAD_STATUS_STAGE_SEEDS): label "Declined Deal", excluded_from_sales = true (closed-lost, like UNQUALIFIED/NOT_SUITABLE — drops out of active sales views but stays on the customer record).

## 4. New email templates
Add both to TEMPLATE_DEFS in `email-templates.js` AND the `email_templates` table so they're editable in the admin email templates page, with {{firstName}} etc. variables:
- `open_deal_deposit_invoice_sent` — subject "Your deposit invoice"; body along the lines of: "Hi {{firstName}}, I've sent over the 10% deposit invoice — please let me know if you haven't received it. Once received, we can then book in a survey visit to confirm the final measurements and design choices."
- `open_deal_declined_thank_you` — subject "Thank you"; body along the lines of: "Hi {{firstName}}, thank you for your time — please feel free to get in touch if you have any questions regarding wardrobes."

## 5. Tests
Add an integration test suite in `/test` following the existing patterns, using QB_API_BASE_OVERRIDE to stub QuickBooks and MAIL_TRANSPORT_FILE_OVERRIDE to capture emails. Cover: deposit = 10% of VAT-inclusive total, estimate accept + others declined, decline flow + optional thank-you, status transitions (OPEN_DEAL stays on amendments, → DEPOSIT_INVOICE on accept, → DECLINED_DEAL on decline), and that a failed invoice send does NOT advance the lead status.
