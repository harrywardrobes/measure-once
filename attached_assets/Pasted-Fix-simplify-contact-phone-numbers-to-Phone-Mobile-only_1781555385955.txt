Fix: simplify contact phone numbers to Phone + Mobile only (remove WhatsApp number field)

The WhatsApp phone-number field is no longer needed anywhere in the app. Every contact
should show only two numbers from HubSpot: Phone (`phone`) and Mobile (`mobilephone`).
Remove the dedicated WhatsApp number field (`hs_whatsapp_phone_number`) from the data
fetching and the UI throughout the application.

IMPORTANT distinction — do NOT remove these (they are unrelated to the number field):
- The "WhatsApp" contact-attempt logging method (call / email / whatsapp) in the Contact
  Customer modal and its server endpoints / DB columns (whatsapp_sent, etc.). That tracks
  *that a WhatsApp message was sent*, not a phone number. Leave it untouched.
- The WhatsApp send feature / Meta API routes (`/api/whatsapp/*`). Leave untouched.

Make these changes (each is idempotent — skip if already done):

1. Server — `server.js`:
   - In every HubSpot contact fetch, remove `hs_whatsapp_phone_number` from the
     `properties` list. Affected handlers include:
       • POST /api/card-actions/contact-customer
       • POST /api/card-actions/arrange-visit
       • GET  /api/contacts/:id  (and any other contact reads)
     New properties lists should request `phone,mobilephone` (plus the other fields they
     already use) but NOT `hs_whatsapp_phone_number`.
   - Remove any line that reads `props.hs_whatsapp_phone_number` (e.g.
     `const whatsapp = String(props.hs_whatsapp_phone_number || '')`,
     `contactWhatsAppPhone`, etc.) and remove those fields from the `res.json({...})`
     payloads. Keep `phone` and `mobile`.
   - Also remove `hs_whatsapp_phone_number` from any `allowed` property update whitelist.
