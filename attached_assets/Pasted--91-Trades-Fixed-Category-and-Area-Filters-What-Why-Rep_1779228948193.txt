#91 - Trades: Fixed Category and Area Filters
What & Why
Replace the free-text search bar in the Trades directory with two fixed-variable filter selectors — Category and Areas Served. These values are stored on the company record (not free text) and let users narrow the list quickly without typing.

Done looks like
The search bar is removed from the Trades section entirely
A Category dropdown replaces it, showing one trade type per company (single select). Options, in alphabetical order:
Carpentry / Roofing
Carpet Fitting
Electrical
Handyman Services
Internal Joinery
Landscaping / Outdoors
Painting + Decorating
Plasterer
Plumbing
An Areas Served multi-select (or chip list) lets a company be assigned more than one area. Options, in alphabetical order:
Anglesey
Chester Only
Cheshire
Greater Manchester
Liverpool
North Wales
Wirral
Wrexham
On the filter/browse side, users pick a single area to narrow results (matching any company whose areas served includes that area)
Both fields are stored on the company record (not free text), replacing any previous freeform trade-type or area fields
Filtering is instant (client-side) with an "All" default that shows every company when no filter is selected
The Add/Edit Company form uses the same fixed lists for these two fields
Out of scope
Free-text search (removed)
Adding new categories or areas via the UI (the lists are code-defined constants)
Any changes outside the Trades directory
Steps
Define constants — Create a shared constants file (or inline in the relevant module) that exports the two ordered arrays: TRADE_CATEGORIES and TRADE_AREAS. These are the single source of truth used by both the form and the filter UI.

Update the company data model — Ensure the company record schema (introduced in the company-first task) stores category as a single string and areasServed as an array of strings, both drawn from the fixed lists.

Update the Add/Edit Company form — Replace any freeform category/area inputs with a single-select dropdown for Category and a multi-select (checkboxes or chip toggles) for Areas Served, both populated from the constants.

Replace the search bar with filter controls — Remove the text search input from the Trades directory header. Add a Category dropdown (includes an "All" option) and an Areas Served dropdown (single pick, includes "All") above the company list.

Wire up client-side filtering — Filter the displayed company list so that: selecting a category shows only companies with that category; selecting an area shows only companies whose areasServed array includes that area; both filters compose (AND logic).
