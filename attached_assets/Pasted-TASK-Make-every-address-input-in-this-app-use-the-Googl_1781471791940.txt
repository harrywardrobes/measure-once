PROBLEM 1 — The integration uses Google's DEPRECATED legacy Places API.
AddressInput.tsx uses google.maps.places.AutocompleteService and
google.maps.places.PlacesService (PlacesService.getDetails). Google no longer enables
the legacy Places API on API keys created after March 2025, so on a modern key the
search box appears but returns ZERO predictions (REQUEST_DENIED) — making every surface
look broken.

FIX: Migrate the client autocomplete to the NEW Places API:
- Replace AutocompleteService with google.maps.places.AutocompleteSuggestion
  (AutocompleteSuggestion.fetchAutocompleteSuggestions).
- Replace PlacesService.getDetails with the new Place class:
  new google.maps.places.Place({ id }).fetchFields({ fields: ['addressComponents'] }).
- Load the library with google.maps.importLibrary('places') (and update
  loadPlacesScript in googleMapsConfig.ts to use the modern async loader, e.g.
  the inline bootstrap loader or ...&loading=async&v=weekly, importing 'places').
- Map the NEW Place addressComponents shape (componentType / longText / shortText)
  to StructuredAddress. NOTE: shared/address.ts googleComponentsToAddress() expects the
  OLD shape (types / long_name / short_name). Add a small adapter so the new
  addressComponents are converted correctly (street_number, route, premise, subpremise,
  postal_town, locality, postal_code, administrative_area_level_1/2, country), or add a
  new mapping function. Keep behaviour identical to today's mapping.
- PRESERVE all existing behaviour: per-surface gating (isAutocompleteEnabled),
  debounce (cfg.autocomplete.debounceMs), minChars, country restriction
  (cfg.autocomplete.countries), language, session tokens (use the new
  AutocompleteSessionToken with fetchAutocompleteSuggestions and pass it through to the
  Place details fetch), the usage beacons via reportGoogleMapsUsage('autocomplete'|'details', ...),
  and graceful silent fallback to manual entry on any error (acFailed path + fallback.mode 'notice').
- Update google-maps.js test-connection to use a Places API (New) endpoint
  (POST https://places.googleapis.com/v1/places:autocomplete) instead of the legacy
  /maps/api/place/autocomplete/json, so the admin "Test connection" reflects reality.
  Keep the usage counters/recent-errors behaviour intact.
