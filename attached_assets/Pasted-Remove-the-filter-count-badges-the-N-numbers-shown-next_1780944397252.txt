Remove the filter-count badges (the (N) numbers shown next to lead-status and sub-status filter chips on the Customers page) — they take up space and aren't needed. This only exists in one place: src/react/pages/CustomersPage.tsx, rendered via the shared FilterChipRow.tsx component.

Important — read before starting: the same count data (store.counts / store.substatusCounts) is used for two different things:

Displaying the (N) badge — this should be removed.
Deciding whether to hide/disable filter chips that currently have zero matching results (e.g. disabled={n === 0} and .filter((s) => (store.counts[s.key] || 0) > 0)) — this behavior should be kept (we still want empty filters hidden/disabled, we just don't want the number shown).
So this is not a simple "delete the counts fetch" — the underlying count data still needs to be fetched to drive visibility/enabled-state, but the display of the number should go away.
