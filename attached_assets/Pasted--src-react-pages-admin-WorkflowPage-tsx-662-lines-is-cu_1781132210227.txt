/src/react/pages/admin/WorkflowPage.tsx (662 lines) is currently a read-only admin reference page that shows which handlers fire per lead status. It needs to become a much richer, fully dynamic developer reference — almost like inline documentation that stays in sync with live configuration.

The page already fetches from six API endpoints on load and has a Refresh button. That data-fetching architecture is correct and should be kept. All new sections must derive from the same six data sources so the page stays accurate as config changes.
